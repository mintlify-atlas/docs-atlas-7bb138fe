## Billing Cycle Mutations

### ChargeBillingCycles

Initiates a bulk charge operation for billing cycles within a date range.

**Variables:**
- `startDate` (DateTime!): Start date for billing cycle range
- `endDate` (DateTime!): End date for billing cycle range
- `contractStatus` ([SubscriptionContractSubscriptionStatus!]): Filter by contract status
- `billingCycleStatus` ([SubscriptionBillingCycleBillingCycleStatus!]): Filter by billing cycle status
- `billingAttemptStatus` (SubscriptionBillingCycleBillingAttemptStatus): Filter by billing attempt status

```graphql
mutation ChargeBillingCycles(
  $startDate: DateTime!
  $endDate: DateTime!
  $contractStatus: [SubscriptionContractSubscriptionStatus!]
  $billingCycleStatus: [SubscriptionBillingCycleBillingCycleStatus!]
  $billingAttemptStatus: SubscriptionBillingCycleBillingAttemptStatus
) {
  subscriptionBillingCycleBulkCharge(
    billingAttemptExpectedDateRange: {
      startDate: $startDate
      endDate: $endDate
    }
    filters: {
      contractStatus: $contractStatus,
      billingCycleStatus: $billingCycleStatus,
      billingAttemptStatus: $billingAttemptStatus
    }
  ) {
    job {
      id
    }
    userErrors {
      field
      message
    }
  }
}
```

### SellingPlanGroupUpdate

Updates a selling plan group and manages its product associations.

**Variables:**
- `id` (ID!): The selling plan group ID
- `input` (SellingPlanGroupInput!): The updated selling plan group data
- `productIdsToAdd` ([ID!]!): Products to add
- `productIdsToRemove` ([ID!]!): Products to remove
- `productVariantIdsToAdd` ([ID!]!): Product variants to add
- `productVariantIdsToRemove` ([ID!]!): Product variants to remove

```graphql
mutation SellingPlanGroupUpdate(
  $id: ID!
  $input: SellingPlanGroupInput!
  $productIdsToAdd: [ID!]!
  $productIdsToRemove: [ID!]!
  $productVariantIdsToAdd: [ID!]!
  $productVariantIdsToRemove: [ID!]!
) {
  sellingPlanGroupUpdate(id: $id, input: $input) {
    sellingPlanGroup {
      id
      sellingPlans(first: 31) {
        edges {
          node {
            ...SellingPlanDetails
          }
        }
      }
    }
    userErrors {
      field
      message
    }
  }
  sellingPlanGroupAddProducts(id: $id, productIds: $productIdsToAdd) {
    userErrors {
      field
      message
    }
  }
  sellingPlanGroupRemoveProducts(id: $id, productIds: $productIdsToRemove) {
    userErrors {
      field
      message
    }
  }
  sellingPlanGroupAddProductVariants(
    id: $id
    productVariantIds: $productVariantIdsToAdd
  ) {
    userErrors {
      field
      message
    }
  }
  sellingPlanGroupRemoveProductVariants(
    id: $id
    productVariantIds: $productVariantIdsToRemove
  ) {
    userErrors {
      field
      message
    }
  }
}
```

<Note>
This mutation uses the `SellingPlanDetails` fragment. See the [Fragments](/api/graphql/fragments) page for details.
</Note>
