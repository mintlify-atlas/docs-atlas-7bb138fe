## Overview

The Shopify Subscriptions Reference App is a comprehensive example application that demonstrates how to build a fully-featured subscription management system for Shopify. Built with Remix, this reference app includes the core functionality of the official Shopify Subscriptions app and serves as both a learning resource and a foundation for building custom subscription solutions.

<Note>
This is a reference implementation provided by Shopify to help developers understand best practices for building subscription apps. It includes production-ready patterns for subscription management, billing, and customer interactions.
</Note>

## What It Does

The Subscriptions Reference App provides a complete subscription management system with:

### Key Features

- **Flexible Subscription Plans**: Create selling plans with customizable billing intervals (weekly, monthly, yearly) and delivery schedules
- **Contract Operations**: Pause, resume, skip, and cancel subscriptions with full webhook integration
- **Payment Management**: Update payment methods and billing addresses for active subscriptions
- **Billing Automation**: Scheduled billing with configurable time zones and retry logic for failed payments
- **Customer Experience**: Theme extensions and checkout UI for seamless subscription purchases
- **Admin Interface**: Built with Polaris for a native Shopify admin experience
- **Multi-Extension Support**: Includes customer account, admin action, POS, and theme extensions

## Architecture

The app is built using modern web technologies and Shopify's recommended patterns:

- **Framework**: [Remix](https://remix.run/) for full-stack React development
- **Database**: Prisma ORM with SQLite (production-ready for PostgreSQL/MySQL)
- **UI**: [@shopify/polaris](https://polaris.shopify.com/) for admin interface components
- **API Client**: [@shopify/shopify-app-remix](https://github.com/Shopify/shopify-app-js) for authentication and API access
- **Extensions**: UI Extensions for customer accounts, admin actions, POS, and themes

### Core Components

```plaintext
app/
├── routes/           # Remix routes for app pages and API endpoints
├── models/           # Data models and business logic
├── components/       # Reusable UI components
├── services/         # External service integrations
└── jobs/             # Background job processing

extensions/
├── buyer-subscriptions/        # Customer account subscription management
├── admin-subs-action/          # Admin subscription quick actions
├── theme-extension/            # Storefront subscription widgets
└── pos-extension/              # Point of sale subscription support
```

## App Store Considerations

<Warning>
If you plan to submit a subscription app to the Shopify App Store, your application must be **substantively different** from this reference app. Simply rebranding or making minor modifications is not sufficient. Your app should provide unique features or solve specific merchant problems that add value beyond the reference implementation.
</Warning>

The Shopify App Store requires apps to:

- Offer unique functionality or serve a specific niche
- Provide clear value differentiation from existing apps
- Meet all [Shopify App Store requirements](https://shopify.dev/docs/apps/store/requirements)
- Follow Shopify's design and UX guidelines

## API Integration

The app integrates with several Shopify APIs:

- **Subscription Contracts API**: Manage subscription contract lifecycle
- **Selling Plans API**: Create and configure subscription selling plans
- **Customer Payment Methods API**: Handle payment method updates
- **Orders API**: Process subscription orders and billing attempts
- **Webhooks**: Real-time notifications for subscription events

<Info>
The app uses Shopify's `unstable` API version for webhooks to access the latest subscription features. For production apps, evaluate which API version best fits your stability requirements.
</Info>
