The customer portal is a Shopify Customer Account UI extension that enables customers to view and manage their subscriptions directly from their account dashboard.

## Extension Configuration

The buyer-subscriptions extension is configured in `shopify.extension.toml`:

```toml
api_version = "2025-01"

[[extensions]]
type = "ui_extension"
name = "Management page"
handle = "buyer-subscriptions"

  [[extensions.targeting]]
  module = "./src/index.tsx"
  target = "customer-account.page.render"

  [extensions.capabilities]
  api_access = true
  network_access = true
```

### Extension Capabilities

<ParamField path="api_access" type="boolean" default="true">
  Enables GraphQL API access for subscription queries and mutations
</ParamField>

<ParamField path="network_access" type="boolean" default="true">
  Allows network requests to the app backend
</ParamField>

<ParamField path="target" type="string" required>
  `customer-account.page.render` - Renders as a customer account page
</ParamField>

## Subscription List

The subscription list displays all customer subscriptions in a responsive grid layout.

### Implementation

```tsx
import {SubscriptionList} from './SubscriptionList';

export function Router() {
  const currentEntry = useNavigationCurrentEntry();
  const url = new URL(currentEntry.url);

  if (url.pathname.includes('subscriptions')) {
    const id = getSubscriptionIdFromPath(url.pathname);
    return <SubscriptionDetails id={id} />;
  }
  
  return <SubscriptionList />;
}
```

## Customer Actions

Self-service actions available to customers through modal dialogs.

### Pause Subscription

<CodeGroup>
```tsx Modal Component
import {PauseSubscriptionModal} from '~/components/PauseSubscriptionModal';

<PauseSubscriptionModal
  contractId={contractId}
  onPauseSubscription={handlePause}
/>
```

```tsx Implementation
async function handlePauseSubscription() {
  setLoading(true);
  const result = await pauseSubscriptionContract(
    PauseSubscriptionContractMutation,
    {
      subscriptionContractId: contractId,
    },
  );

  if (result?.subscriptionContractPause?.contract?.status !== 'PAUSED') {
    setError(true);
    if (result?.subscriptionContractPause?.userErrors?.length) {
      setErrorMessage(
        result?.subscriptionContractPause?.userErrors[0].message,
      );
    }
    setLoading(false);
    return;
  }

  onPauseSubscription();
  sendRequest({contractId, operationName: 'PAUSE'});
}
```

```graphql Mutation
mutation PauseSubscriptionContract($subscriptionContractId: ID!) {
  subscriptionContractPause(
    subscriptionContractId: $subscriptionContractId
  ) {
    contract {
      id
      status
    }
    userErrors {
      field
      message
    }
  }
}
```
</CodeGroup>

### Resume Subscription

```tsx
import {ResumeSubscriptionModal} from '~/components/ResumeSubscriptionModal';

<ResumeSubscriptionModal
  contractId={contractId}
  onResumeSubscription={() => {
    showSuccessToast(SuccessToastType.Resumed);
    refetchSubscriptionContract();
  }}
/>
```

**Resume Flow:**
1. Customer clicks "Resume" in details actions
2. Modal opens with confirmation message
3. `subscriptionContractActivate` mutation executes
4. Status validates as `ACTIVE`
5. Success toast displays
6. Subscription data refetches

### Cancel Subscription

```tsx
import {CancelSubscriptionModal} from '~/components/CancelSubscriptionModal';

<CancelSubscriptionModal
  contractId={contractId}
  onCancelSubscription={() => {
    showSuccessToast(SuccessToastType.Cancelled);
    navigate('extension:/');
  }}
/>
```

<Warning>
Cancellation is permanent. The modal includes a confirmation step to prevent accidental cancellations.
</Warning>

### Skip Next Order

```tsx
import {SkipNextOrderModal} from '~/components/SkipNextOrderModal';

<SkipNextOrderModal
  contractId={contractId}
  cycleIndexToSkip={nextBillingCycle.cycleIndex}
  resumeDate={nextBillingCycle.billingAttemptExpectedDate}
  onSkipOrder={() => {
    showSuccessToast(SuccessToastType.Skipped);
    refetchSubscriptionContract();
  }}
/>
```

**Skip Features:**
- Shows resume date after skipped order
- Validates billing cycle skipped status
- Updates upcoming orders list
- Displays formatted date in customer timezone

```tsx
function SkipNextOrderModal({resumeDate, cycleIndexToSkip}) {
  const formatDate = useFormatDate();
  
  async function handleSkipNextOrder() {
    const result = await skipBillingCycle(SkipBillingCycleMutation, {
      subscriptionContractId: contractId,
      billingCycleIndex: cycleIndexToSkip,
    });

    if (!result?.subscriptionBillingCycleSkip?.billingCycle?.skipped) {
      // Handle error
      return;
    }

    onSkipOrder();
  }
  
  return (
    <TextBlock>
      {resumeDate
        ? i18n.translate('skipNextOrder.date', {
            date: formatDate(resumeDate),
          })
        : i18n.translate('skipNextOrder.generic')}
    </TextBlock>
  );
}
```

## Error Handling

All customer actions include comprehensive error handling:

```tsx
const [error, setError] = useState(false);
const [errorMessage, setErrorMessage] = useState<string | undefined>();

// In mutation handler
if (result?.subscriptionContractPause?.userErrors?.length) {
  setError(true);
  setErrorMessage(
    result?.subscriptionContractPause?.userErrors[0].message,
  );
  return;
}

// In UI
{error ? (
  <Banner
    status="critical"
    title={i18n.translate('pauseSubscriptionModal.errorBannerTitle')}
  >
    {errorMessage}
  </Banner>
) : null}
```
