## Get Started in Minutes

This quickstart guide will walk you through cloning, running, and creating your first subscription plan with the Shopify Subscriptions Reference App.

<Steps>
  <Step title="Clone and Install">
    Use the Shopify CLI to create a new app from the template:
    
    <CodeGroup>
    ```bash Shopify CLI
    shopify app init --template https://github.com/Shopify/subscriptions-reference-app
    cd subscriptions-reference-app
    ```
    
    ```bash Manual Clone
    git clone https://github.com/Shopify/subscriptions-reference-app.git
    cd subscriptions-reference-app
    ```
    </CodeGroup>
    
    Install dependencies:
    
    <CodeGroup>
    ```bash pnpm
    pnpm install
    ```
    
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Set Up Database">
    Initialize the SQLite database with Prisma:
    
    ```bash
    pnpm setup
    ```
    
    This command runs:
    - `prisma generate` - Generates Prisma Client
    - `prisma migrate deploy` - Applies database migrations
    
    <Info>
      The app uses SQLite by default for development. For production, configure PostgreSQL or MySQL in your environment variables.
    </Info>
  </Step>
  
  <Step title="Start the Development Server">
    Run the app with the Shopify CLI:
    
    ```bash
    pnpm dev
    ```
    
    This will:
    1. Start the Remix development server
    2. Open a tunnel for local development
    3. Prompt you to select a Partner organization and development store
    4. Install the app on your development store
    
    <Note>
      You'll need a [Shopify Partner account](https://partners.shopify.com) and a development store to test the app.
    </Note>
  </Step>
  
  <Step title="Access the App">
    After installation, the CLI will provide a URL to access your app in the Shopify admin:
    
    ```
    https://admin.shopify.com/store/YOUR-STORE/apps/subscriptions-app
    ```
    
    Click through to open the app embedded in your Shopify admin.
  </Step>
  
  <Step title="Create Your First Selling Plan">
    Now let's create a subscription plan for your products:
    
    1. **Navigate to Plans**: In the app, click "Create Plan" or go to the Plans section
    
    2. **Configure Basic Details**:
       ```
       Plan Name: Monthly Coffee Subscription
       Merchant Code: monthly-coffee
       ```
    
    3. **Set Delivery Options**:
       - Delivery Frequency: Every 1 Month
       - Discount: 10% off
    
    4. **Select Products**: Choose which products customers can subscribe to
    
    5. **Save the Plan**: Click "Save" to create your selling plan group
    
    <Tip>
      You can create multiple delivery options (weekly, monthly, quarterly) in a single selling plan group.
    </Tip>
  </Step>
  
  <Step title="Test a Subscription Purchase">
    Now test the customer experience:
    
    1. **Visit Your Store**: Go to your development store's online storefront
    
    2. **Add a Subscription Product**: Find a product you added to the selling plan
    
    3. **Select Subscription**: On the product page, you'll see subscription options via the admin extension
    
    4. **Complete Checkout**: Use a test payment method:
       ```
       Card: 4242 4242 4242 4242
       Expiry: Any future date
       CVC: Any 3 digits
       ```
    
    5. **View the Contract**: After purchase, return to the app to see the new subscription contract
  </Step>
  
  <Step title="Manage the Subscription">
    Explore subscription management features:
    
    - **View Contract Details**: Click on the contract to see customer info, billing schedule, and line items
    - **Pause Subscription**: Test pausing the subscription
    - **Skip Billing Cycle**: Skip an upcoming billing attempt
    - **Update Delivery Address**: Change the shipping address
    - **Cancel Subscription**: Cancel the subscription
    
    All these actions are available in the contract details page.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="App won't start - port already in use">
    If you see `EADDRINUSE` error, another process is using port 3000. Either:
    - Kill the existing process: `lsof -ti:3000 | xargs kill -9`
    - Or change the port in `vite.config.ts`
  </Accordion>
  
  <Accordion title="Database errors during setup">
    If Prisma migrations fail:
    ```bash
    rm -rf prisma/data.db
    pnpm setup
    ```
    This recreates the database from scratch.
  </Accordion>
  
  <Accordion title="Can't see subscription options on product page">
    Make sure:
    1. Your selling plan group includes the product or variant
    2. The admin extension is properly deployed (check Extensions in Shopify CLI output)
    3. Clear your browser cache and refresh the page
  </Accordion>
  
  <Accordion title="GraphQL errors when creating selling plans">
    Verify your app has the required scopes in `shopify.app.toml`:
    - `write_products`
    - `write_own_subscription_contracts`
    - `write_customer_payment_methods`
  </Accordion>
</AccordionGroup>
